Spring框架概述

February 23, 2023 6845

本文概述Spring Framework主要模块，并提供了对应的Example，阅读完此文可对Spring有个大致的了解。

1. 前言

Spring是一个庞大的项目，其包含了许多子技术（框架）。

本文主要概述Spring Framework部分。

2. Spring的诞生

最初Java开发推崇EJB为核心的方式开发，但是这种开发方式存在各种弊端。

例如，使用复杂，代码侵入性强，开发周期长，可移植性差等。

Rod Johnson 2002年编著的一本《Expert One-on-One J2EE Design and Development》中，针对 以EJB为核心的开发进行了逐一分析和否定，并对此提供了更加优异的方式进行了替换。

在书中Rod Johnson提供了一个3W行代码的附件。其展示了如何在不使用EJB的情况下，构建一个高质量，高可用，可扩展，移植性高的Java应用。在此书附件中，提供了上万行的基础代码，其中包含了许多类和接口，其中让人悉知的有ApplicationContext，BeanFactory等。

此书一出，引起巨大反响。不久后Rod Johnson将附件中的代码开源。并把此附件代码命名为Spring，其含义为”Spring如春风一般，带走了Java开发的寒冬“。

此处的Spring框架尤指Spring Framework，是Spring项目的核心。其它子项目都是围绕着此核心而扩展的。

附件中代码的包名为：com.interface21，其含义是作为21世纪编码的参考

3. Spring Framework概述

Spring Framework框架拥有许多个子模块，但主要由几个核心的子模块组成，其它的子模块都是可选的。

核心的子模主要由：spring-core ，spring-beans，spring-context， spring-expression构成。由于Spring官方调整，后来的core，beans，expression已经被内嵌进context模块中了。也就是说只需添加context模块依赖就可以使用Spring Framework了。

3.1 core 模块

此模块提供了IOC功能，提供XML与注解的使用方式。

IOC也称之为DI（依赖注入）。

3.1.1 IOC

Inversion of Control（IOC），翻译过来就是控制反转。

控制反转简单诠释就是：将对Bean的管理权限交给Spring。

为什么要把对Bean的控制权限交给Spring呢？有什么好处呢？

  1. Spring提供了许多非常实用，新颖的技术。

  2. Spring不会控制没有控制权限的class，没有控制权限的class使用不了Spring提供的任何功能。

  3. 因为Spring控制的class默认是单实例，所以将class控制权限交给Spring可以很大层面上节约不必要的内存消耗。

  4. Spring是Java开发中非常重要，且常用的一个框架。庞大的生态体系几乎包含了Java大部分开发场景。

4. Spring 特点

4.1 代码侵入性低

4.2 方便解耦，简化开发

得益于IOC与DI，很大程度上避免了程序硬编码造成的高耦合。

4.3 AOP编程的支持

切面编程可以将大量重复的代码复用，减少无用代码的冗余。

4.4 声明式事务的支持

得益于Spring的数据访问模块，使得可以声明式进行事务的控制。

4.5 方便程序的测试

Spring提供了测试模块，其可以在测试时更加方便。

4.6 方便集成各种优秀框架

大部分优秀的框架都提供了Spring的集成（starter）。

4.7 降低JavaEE API使用难度

Spring web模块的功能能够满足大部分开发场景，封装了大部分复杂的接口，提供了更加简洁的API。
